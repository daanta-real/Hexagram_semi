### Hexagram Semi Project

### 팀명 : <b>Hexagram</b>

### 주제 : 관광장소 안내 및 리뷰 공유 웹사이트 제작

##### 사이트 제작 목적 : 코로나 시대의 해외여행 기회 단절과 국내여행에 대한 수요 증가.
##### 사용자들이 노가리 까먹듯이 편하게 여행지를 공유하고 즐길 수 있는 공간을 만들기 위한 목적으로 제작

## 주요 구성 항목

- 프로젝트 소개
  - 애플리케이션 개발 목적
  - 인원 구성 및 역할 분담
  - 개발 환경
  - 개발 일정
  - 시장 조사 내용
- 다이어그램
  - 데이터베이스 흐름도(ERD)
  - 유스케이스 다이어그램
  - 시퀀스 다이어그램
  - 클래스 다이어그램
  - 계층별 사이트맵(컨텐츠 분류 또는 플로우 그래프)
- 점검 체크리스트 및 수행 결과
  - 성능 테스트 수행 결과
  - 사용성 테스트 수행 결과
- 핵심 장면 스크린샷
- 한계점, 보완사항 및 계획, 느낀점
- 이외 추가적인 내용들을 자율적으로 추가

### 사용 기술 스택 / 사용 언어

- Operating System: Windows 10
- BE(Back-End): Java SE 1.8 / Apache Tomcat 9.0
- FE(Front End): HTML 5 / JavaScript ES 5 / JQuery 3.6 / CSS 2.1
- DBMS: Oracle RDBMS (Oracle Database 11g)
- Developer tool: Eclipse 2020-06(Java, FE) / SQL-Developer 18 (Oracle) / VScode 1.62 (FE)

### 사용한 오픈 API
  - 카카오 주소 찾기 API
  - Daum 우편 번호 API

### 프로젝트 기능 구현
-  **박준성/민선아(공동작업)**
    - 회원관련 CRUD 기본 기능 구현, Login/Admin Filter, Sessioner 등 권한 검사 강화 작업
    - 각 목록 페이지에 사용할 공통 Pagination 모듈 제작 
    - 회원 비밀번호 암호화 저장☆ 
    - 회원은 웹사이트 내에서도 절대 오류가 나면 안되는 파트이기 때문에 각종 검사 기능에 집중 또 집중
    - Users table 제약조건 정규식 검사 방식 연구 및 등록값 검사  
    - Class Diagram / Usecase Diagram / Users Flow Chart 제작

-  **김의동/홍성진(공동작업)**
    - 코스 및 관광지 조회 페이지 구현 및 각 페이지 JS 및 CSS 작업
    - 코스 및 관광지 CRUD 구현 및 계층형 댓글 구현
    - 코스 및 관광지 필터 작업 및 오류 수정작업
    - 해당 부분 플로우 차트 및 문서 작업 시행
  
-  **김소은/조수현(공동작업)**
    - 게시판의 기초 기능인 CRUD 구현에 역량을 집중
    - 이벤트, 이벤트 첨부파일 DTO/DAO
    - 이벤트 소개글 작성/조회/수정/삭제 페이지
    - 이벤트 목록 및 단일 컬럼 검색페이지 구현
